# 股票资讯页面 - 使用指南

## 📍 页面位置

http://localhost:3000 → **股票资讯** 标签页

---

## 🎯 主要功能

### 1. Profile 数据填充进度 (顶部卡片)

显示系统对所有 2990 个新闻股票的 Profile 数据填充情况。

```
📋 Profile 数据填充进度
✅ 已完成: 3 / 2990  📊 完成度: 0.10%  📈 平均: 0.10%
[==] (进度条)
```

**含义**:
- **已完成**: 3 个股票的 Profile 完成度 ≥50%
- **完成度**: 3/2990 = 0.10%
- **平均**: 所有 2990 个股票的平均完成度

---

### 2. 股票列表 (下方表格)

展示所有出现在新闻中的股票，按 **Profile 完成度排序**（已完成的优先显示）。

#### 表格列说明

| 列名 | 说明 | 示例 |
|------|------|------|
| **名称** | 公司简称 | 平安银行、比亚迪 |
| **代码** | 股票代码 | 000001.SH, 002594.SZ |
| **开始统计** | 该股票的资讯统计开始日期 | 2020-11-01 |
| **文章数** | 该股票相关的新闻文章数 | 293, 235, 5 |
| **Profile 完成度** ⭐ **新** | Profile 信息的完成度百分比 + 进度条 | `[====] 50%` |
| **更新状态** | 是否已完成 Profile 更新 | 🟢 (已完成) / ⚪ (待更新) |
| **最后更新** | 上次更新 Profile 的时间 | 2025-08-28 |
| **操作** | 查看详情 | [查看] 按钮 |

## 🎯 Profile 完成度说明

### 9 个字段

Profile 包含 9 个字段：
1. 行业 (industry)
2. 业务概述 (business_summary)
3. 核心产品 (core_products)
4. 竞争地位 (competitive_position)
5. 竞争对手 (competitors)
6. 战略关键词 (strategic_keywords)
7. 风险因素 (risk_factors)
8. 历史亮点 (history_highlights)
9. 完整 JSON (profile_json)

### 完成度计算

```
完成度 = (已填充字段数 / 9) × 100%

例如:
- 已填充 5 个字段: 完成度 = 5/9 ≈ 56% ✅ 已完成（≥50%）
- 已填充 3 个字段: 完成度 = 3/9 ≈ 33% ⚪ 待更新（<50%）
- 已填充 0 个字段: 完成度 = 0/9 = 0% ⚪ 待更新
```

### 完成状态

- **✅ 已完成**: 完成度 ≥50%（至少 5 个字段已填充）
- **⚪ 待更新**: 完成度 <50%（不足 5 个字段填充）

## ❓ 常见问题

### Q: 为什么有些股票显示 0%？
A: 这些股票的 Profile 数据还没有被系统填充。系统定时任务会自动更新这些股票，预计 60 天内完成全部 2990 只股票。

### Q: 为什么排序会改变？
A: 当新的 Profile 更新完成时，完成度会增加，列表会根据新的完成度重新排序。已完成的股票会自动移到前面。

### Q: 如何加快 Profile 更新速度？
A: 系统自动定时更新，目前设置为每周一 02:00 执行一次。如需手动触发，请联系管理员。

### Q: Profile 完成度是否会实时更新？
A: 不是实时的，系统每 30 秒检查一次服务器是否有新的更新数据，然后刷新页面显示。
