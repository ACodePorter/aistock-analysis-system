## 🎯 解决方案

### 1. 优先显示已完成的股票

**改变**: 在 `load()` 函数中添加排序逻辑

```typescript
// 按完成度排序（已完成优先，即完成度降序）
itemsWithCompletion.sort((a: any, b: any) => 
  (b.completion_percentage ?? 0) - (a.completion_percentage ?? 0)
)
```

**效果**: 
- 完成度 ≥50% 的股票显示在最前面
- 完成度 <50% 的股票在后面
- 列表会自动按此顺序显示

### 2. 修复更新状态判断逻辑

**问题**: 旧代码使用 `is_updated` 字段判断，该字段不准确

**解决**:
```typescript
// ❌ 旧逻辑
backgroundColor: it.is_updated ? '#10b981' : '#d1d5db'

// ✅ 新逻辑
const isCompleted = (it.completion_percentage ?? 0) >= 50
backgroundColor: isCompleted ? '#10b981' : '#d1d5db'
```

**效果**:
- 根据 **Profile 完成度** 判断（≥50% = 已完成）
- 绿色圆点准确反映 Profile 完成状态
- 与后端定时任务的完成度判断逻辑一致

### 3. 简化 Profile 进度卡片

**前**: 显示大网格卡片，每个股票一个完成度进度条

**后**: 
- 简化为一个小的总体统计卡片
- 只显示总体数据：总股票数、已完成数、完成度百分比
- 移除网格卡片显示

**新卡片样式**:
```
📋 Profile 数据填充进度
✅ 已完成: 3 / 2990  📊 完成度: 0.10%  📈 平均: 0.10%
[===========================]  (简单进度条)
```

## 🔧 技术修改

### 文件: `frontend/src/ui/StocksNewsIndex.tsx`

| 修改 | 说明 |
|-----|------|
| 接口扩展 | `StockItem` 添加 `completion_percentage`, `fields_filled`, `total_fields` |
| 数据加载 | 合并 `load()` 和 `loadProgress()` 的数据 |
| 数据排序 | 按 `completion_percentage` 降序排序 |
| 状态判断 | 从 `is_updated` 改为 `completion_percentage >= 50` |

| 新增列 | 添加 "Profile 完成度" 列 |

### 代码结构变化

```typescript
// 旧逻辑
load() → 获取列表
loadProgress() → 获取进度（分离）

// 新逻辑
load() → 同时获取列表 + 进度
  ↓
  合并数据（为每个股票添加完成度）
  ↓
  按完成度排序
  ↓
  渲染表格
```

## 📊 数据流

```
后端 API
├─ GET /api/news/stocks (列表)
│  └─ 返回: StockItem[]
└─ GET /api/news/stocks/progress (进度)
   └─ 返回: {total_stocks, completed_profiles, stocks_detail[]}

前端
├─ load() 合并两个 API 返回值
├─ 为每个 StockItem 添加: completion_percentage, fields_filled
├─ 按 completion_percentage 排序
└─ 渲染表格 (已完成的优先显示)
```
