🚀 工作原理

1️⃣ 应用启动
   └─ 自动初始化后台任务调度器

2️⃣ 每周一 02:00
   └─ 后台任务自动触发

3️⃣ 股票收集和排序
   └─ 从新闻中提取 2990 个股票
   └─ 按完成度排序（低优先）

4️⃣ 批量处理
   └─ 逐个调用 LLM 分析
   └─ 每个股票间隔 2 秒
   └─ 更新数据库

5️⃣ 日志和报告
   └─ 记录所有执行信息
   └─ 生成执行统计报告

🔧 配置方式

执行时间修改（第 120 行）:
  CronTrigger(day_of_week=0, hour=2, minute=0)  # 周一 02:00
  CronTrigger(hour=23, minute=0)  # 每天 23:00
  CronTrigger(day_of_week=4, hour=10, minute=0)  # 周五 10:00

速率控制修改（第 148 行）:
  delay_between_stocks=2.0  # 默认 2 秒
  delay_between_stocks=5.0  # 更保险（5 秒）
  delay_between_stocks=1.0  # 更快（1 秒）

批处理数量修改（返回语句）:
  return incomplete_stocks  # 全部处理
  return incomplete_stocks[:500]  # 只处理 500 个
  return incomplete_stocks[:100]  # 只处理 100 个

🛠️ 故障排除

问题：任务没有自动运行
解决：
  1. 检查应用启动日志是否包含 "后台任务调度器已初始化"
  2. 检查 APScheduler 是否安装 (pip list | grep apscheduler)
  3. 检查下次执行时间是否已设置

问题：任务运行失败
解决：
  1. 查看错误日志 (grep "❌" logs/app.log)
  2. 检查 OpenAI API 密钥是否正确
  3. 检查数据库连接是否正常

问题：任务运行太慢
解决：
  1. 增加延迟时间到 5 秒以防超时
  2. 减少一次处理的股票数量到 500 或更少
  3. 检查网络和 LLM 服务是否正常

💡 常见问题

Q: 是否需要手动启动任务？
A: 不需要，应用启动时会自动初始化

Q: 任务何时执行？
A: 每周一凌晨 02:00（可修改配置）

Q: 是否会影响应用性能？
A: 不会，任务在后台单独线程运行

Q: 如何验证任务是否正在运行？
A: 查看日志或实现状态查询 API

Q: 是否可以修改执行时间？
A: 可以，编辑 task_scheduler.py 第 120 行的 CronTrigger
